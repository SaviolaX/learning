## Рівень 1 — базові канали

### 1. Pipeline обробки чисел ✅
Три горутини з'єднані каналами: generate → square → print. 
Класичний pipeline патерн. Навчить як дані течуть між горутинами.

### 2. Fan-out / Fan-in
Один канал з задачами → N горутин-воркерів обробляють паралельно → один канал збирає результати. 
Саме те що ти робив з URL checker, але усвідомлено.

## Рівень 2 — select і таймаути

### 1. Rate limiter
Обмежити кількість запитів до N на секунду використовуючи time.Ticker і select. 
Реальна задача — використовується скрізь в продакшн коді.

### 2. Timeout wrapper
Функція яка виконує будь-яку операцію, але якщо вона не завершилась за N секунд — повертає помилку. 
Тільки select, time.After і канали.

## Рівень 3 — координація горутин

### 1. Worker pool ✅
Черга задач + фіксована кількість воркерів + graceful shutdown. 
Навчить sync.WaitGroup в комбінації з каналами.

### 2. Pub/Sub брокер (мін-версія того що ти вже зробив)
Але цього разу з топіками: підписники реєструються на конкретний топік, 
повідомлення йдуть тільки до потрібних підписників.

## Рівень 4 — складніші патерни

### 1. Circuit breaker
Якщо сервіс падає — автоматично перестати до нього звертатись на певний час. 
Чистий канальний стейт-машин.

### 2. Crawler з обмеженням паралельності
Обходить сторінки по посиланнях, але не більше N одночасних запитів. 
Комбінація semaphore паттерну + канали + контекст.
